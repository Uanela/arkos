## What you'll build

By the end of this tutorial you'll have a fully working Bible verses API — the kind a devotional app, church website, or scripture study tool would actually use.

Clients will be able to browse books, chapters, and verses publicly. Authenticated users will be able to save favourite verses with personal notes and attach audio recordings to any verse.

## What you'll learn

Working through each chapter in order you will:

- Scaffold a project with `create-arkos`
- Model data with Prisma and push it to PostgreSQL
- Get full CRUD on every model for free — zero boilerplate
- Protect routes with `ArkosPolicy` and `RouteHook`
- Inject business logic with controller interceptors
- Validate request bodies with generated Zod schemas
- Build a custom passage-lookup endpoint
- Handle audio file uploads on a route
- Browse and test every endpoint at `/api/docs`

## Who this is for

This tutorial assumes you know TypeScript and have used an HTTP API before. You do not need prior experience with Arkos, Prisma, or Express.

## Prerequisites

Make sure you have these installed before you start:

- Node.js 22.9 or later
- pnpm (`npm install -g pnpm`)
- A running SQLite database
