## Scaffold the project

Run `create-arkos` and answer the prompts exactly as shown below:

```bash
pnpm create arkos@latest
```

```
? What is the name of your project? › bible-api
? Would you like to use TypeScript? › Yes
? What db provider will be used for Prisma? › postgresql
? Would you like to set up Validation? › Yes
? Choose validation library › zod
? Would you like to set up Authentication? › Yes
? Choose authentication type › static
? Choose default username field for login › email
? Would you like to use authentication with Multiple Roles? › Yes
```

`create-arkos` creates the project folder, installs dependencies, and runs `prisma generate` automatically.

## Environment variables

Open `.env` and fill in your database URL and a random JWT secret:

```bash title=".env"
DATABASE_URL=postgresql://user:pass@localhost:5432/bible_api
JWT_SECRET=change-me-to-something-long-and-random
PORT=8000
```

## Start the dev server

```bash
cd bible-api
pnpm run dev
```

Your API is already live at `http://localhost:8000/api` — with CORS, rate limiting, compression, and security headers wired up out of the box.

<Callout type="info">
  You can test any endpoint at `http://localhost:8000/api/docs` once you add the OpenAPI config in Chapter 9. Keep that URL in mind — it will be your main testing tool throughout the tutorial.
</Callout>

## What got created

```
bible-api/
├── prisma/
│   └── schema/
│       └── schema.prisma      # Your Prisma schema
├── src/
│   ├── modules/
│   │   └── user/              # Auth module — generated by create-arkos
│   ├── utils/
│   │   └── prisma.ts          # Prisma client singleton
│   └── app.ts                 # Entry point
├── arkos.config.ts            # Arkos configuration
└── .env
```

The `user` module is fully functional already — registration, login, and JWT auth work without any extra code.
