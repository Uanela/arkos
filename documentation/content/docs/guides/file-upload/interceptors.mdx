Arkos's file upload routes support the same interceptor system as Prisma model routes and authentication routes — run logic before or after any file operation without replacing the built-in behavior.

<Callout type="info">
This guide covers interceptors for the standalone file upload routes. For file uploads on `ArkosRouter` or via `RouterConfig`, see [File Upload](/docs/guides/file-upload/router-upload).
</Callout>

## Available Interceptors

| Endpoint | Interceptors |
|---|---|
| `POST /api/uploads/:fileType` | `beforeUploadFile`, `afterUploadFile` |
| `PATCH /api/uploads/:fileType/:fileName` | `beforeUpdateFile`, `afterUpdateFile` |
| `DELETE /api/uploads/:fileType/:fileName` | `beforeDeleteFile`, `afterDeleteFile` |
| `GET /api/uploads/:fileType/:fileName` | `beforeFindFile` |

## File Structure
```
src/modules/file-upload/
├── file-upload.middlewares.ts   # reusable functions
└── file-upload.interceptors.ts  # interceptor chains
```

Generate them with the CLI:
```bash
arkos generate interceptors --module file-upload
# or
arkos g i -m file-upload
```

## Examples

### Before — Validate File Is Not Referenced Before Deleting
```ts title="src/modules/file-upload/file-upload.middlewares.ts"
import { ArkosRequest, ArkosResponse, ArkosNextFunction } from "arkos";
import { AppError } from "arkos/error-handler";

export const checkFileReferences = async (
  req: ArkosRequest,
  res: ArkosResponse,
  next: ArkosNextFunction
) => {
  const { fileName } = req.params;
  const isReferenced = await fileReferenceService.check(fileName);

  if (isReferenced)
    throw new AppError(
      "Cannot delete file: still referenced in database.",
      400,
      {},
      "FileStillReferenced"
    );

  next();
};
```
```ts title="src/modules/file-upload/file-upload.interceptors.ts"
import { checkFileReferences } from "./file-upload.middlewares";

export const beforeDeleteFile = [checkFileReferences];
```

### After — Log Upload Activity
```ts title="src/modules/file-upload/file-upload.middlewares.ts"
import { ArkosRequest, ArkosResponse, ArkosNextFunction } from "arkos";

export const logUpload = async (
  req: ArkosRequest,
  res: ArkosResponse,
  next: ArkosNextFunction
) => {
  const urls = res.locals.data.urls;

  await activityLog.record({
    userId: req.user?.id,
    action: "file_upload",
    files: urls,
  });

  next();
};
```
```ts title="src/modules/file-upload/file-upload.interceptors.ts"
import { logUpload } from "./file-upload.middlewares";

export const afterUploadFile = [logUpload];
```

### Before — Track File Access
```ts title="src/modules/file-upload/file-upload.middlewares.ts"
import { ArkosRequest, ArkosResponse, ArkosNextFunction } from "arkos";

export const trackFileAccess = async (
  req: ArkosRequest,
  res: ArkosResponse,
  next: ArkosNextFunction
) => {
  await accessLog.record({
    fileName: req.params.fileName,
    userId: req.user?.id,
    accessedAt: new Date(),
  });

  next();
};
```
```ts title="src/modules/file-upload/file-upload.interceptors.ts"
import { trackFileAccess } from "./file-upload.middlewares";

export const beforeFindFile = [trackFileAccess];
```

## Execution Order
```
Request → beforeX → Core File Upload Logic → afterX → Response
```

After interceptors access the result via `res.locals.data` — for uploads this includes the `urls` array of the uploaded files.

## Configuring File Upload Routes

Beyond interceptors, each file upload endpoint accepts the same configuration object as `ArkosRouter`. See [File Upload Routes](/docs/guides/file-upload/routes) for available operations and configuration options.
