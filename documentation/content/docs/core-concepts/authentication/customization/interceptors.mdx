Arkos's built-in authentication endpoints support the same interceptor system as Prisma model routes — run logic before, after, or on error of any auth operation without replacing the built-in behavior.

<Callout type="warning">
Before using authentication interceptors, make sure you have authentication configured. See [Authentication Setup](/docs/core-concepts/authentication/setup).
</Callout>

## Available Authentication Interceptors

| Route | Interceptors |
|---|---|
| `POST /api/auth/login` | `beforeLogin`, `afterLogin`, `onLoginError` |
| `POST /api/auth/signup` | `beforeSignup`, `afterSignup`, `onSignupError` |
| `DELETE /api/auth/logout` | `beforeLogout`, `afterLogout`, `onLogoutError` |
| `POST /api/auth/update-password` | `beforeUpdatePassword`, `afterUpdatePassword`, `onUpdatePasswordError` |
| `GET /api/users/me` | `beforeGetMe`, `afterGetMe`, `onGetMeError` |
| `PATCH /api/users/me` | `beforeUpdateMe`, `afterUpdateMe`, `onUpdateMeError` |
| `DELETE /api/users/me` | `beforeDeleteMe`, `afterDeleteMe`, `onDeleteMeError` |

## File Structure

Interceptors live in `src/modules/auth/`:
```
src/modules/auth/
├── auth.middlewares.ts   # reusable functions
└── auth.interceptors.ts  # interceptor chains
```

Generate them with the CLI:
```bash
arkos generate interceptors --module auth
# or
arkos g i -m auth
```

## Examples

### Before — Rate Limiting Login Attempts
```ts title="src/modules/auth/auth.middlewares.ts"
import { ArkosRequest, ArkosResponse, ArkosNextFunction } from "arkos";
import { AppError } from "arkos/error-handler";

export const trackLoginAttempts = async (
  req: ArkosRequest,
  res: ArkosResponse,
  next: ArkosNextFunction
) => {
  const identifier = req.body.email || req.body.username;
  const attempts = await getFailedLoginAttempts(identifier);

  if (attempts > 5)
    throw new AppError("Too many failed attempts. Try again later.", 429);

  next();
};
```
```ts title="src/modules/auth/auth.interceptors.ts"
import { trackLoginAttempts } from "./auth.middlewares";

export const beforeLogin = [trackLoginAttempts];
```

### After — Send Verification Email on Signup
```ts title="src/modules/auth/auth.middlewares.ts"
import { ArkosRequest, ArkosResponse, ArkosNextFunction } from "arkos";

export const sendVerificationEmail = async (
  req: ArkosRequest,
  res: ArkosResponse,
  next: ArkosNextFunction
) => {
  const user = res.locals.data.data;

  emailService
    .sendVerificationEmail(user.email, user.verificationToken)
    .catch(console.error);

  next();
};
```
```ts title="src/modules/auth/auth.interceptors.ts"
import { sendVerificationEmail } from "./auth.middlewares";

export const afterSignup = [sendVerificationEmail];
```

### Error — Increment Failed Attempts on Login Error
```ts title="src/modules/auth/auth.middlewares.ts"
import { ArkosRequest, ArkosResponse, ArkosNextFunction } from "arkos";

export const incrementFailedAttempts = async (
  err: any,
  req: ArkosRequest,
  res: ArkosResponse,
  next: ArkosNextFunction
) => {
  const identifier = req.body.email || req.body.username;
  if (identifier) await incrementFailedLoginAttempts(identifier);
  next(err);
};
```
```ts title="src/modules/auth/auth.interceptors.ts"
import { incrementFailedAttempts } from "./auth.middlewares";

export const onLoginError = [incrementFailedAttempts];
```

## Execution Order
```
Request → beforeX → Core Auth Logic → afterX → Response
                          ↓ (on failure)
                        onXError
```

After interceptors access the result via `res.locals.data.data`. For login and signup, `res.locals.additional.user` holds the raw user object before it's shaped for the response.

## Validation

Auth endpoints also support request validation — custom schemas or DTOs for login, signup, update-password, and me routes. See [Authentication Validation](/docs/core-concepts/authentication/customization/validation) for full configuration.

## Configuring Authentication Routes

Beyond interceptors, each auth endpoint accepts the same configuration object as `ArkosRouter` — disable routes, add rate limiting, and more:
```ts title="src/modules/auth/auth.router.ts"
import { ArkosRouter, RouterConfig } from "arkos";

export const config: RouterConfig = {
  login: {
    rateLimit: { windowMs: 15 * 60_000, max: 10 },
  },
  signup: {
    disabled: true,
  },
};

const router = ArkosRouter();

export default router;
```

See [Configuring Authentication Routes](/docs/core-concepts/authentication/routes#configuring-authentication-routes) for the full reference.
